# Прокси для обхода блокировок API

В некоторых регионах (например, в РФ) прямой доступ к `generativelanguage.googleapis.com` (Gemini) и `nano-gpt.com` (NanoGPT) заблокирован. Этот Cloudflare Worker — прозрачный прокси, который разворачивается за минуту и бесплатно (в рамках лимитов Cloudflare).

## Что делает воркер

Проксирует два вышестоящих API по префиксу пути:

| Префикс    | Вышестоящий API                          | Кому нужно                    |
|------------|------------------------------------------|-------------------------------|
| `/gemini/*`| `https://generativelanguage.googleapis.com` | Бесплатный режим (свой ключ Gemini) |
| `/nano/*`  | `https://nano-gpt.com`                   | Режимы «Простой» / «Продвинутый» (NanoGPT) |

Ключи **не хранятся** в воркере — клиент передаёт их в заголовках, воркер лишь перенаправляет запрос и ответ.

## Быстрый старт (через Dashboard)

1. Зайдите на https://dash.cloudflare.com → **Workers & Pages** → **Create** → **Create Worker**.
2. Дайте имя (например `nutrilens-proxy`), нажмите **Deploy**.
3. Откройте созданный воркер → **Edit code**.
4. Скопируйте содержимое `worker.js` из этой папки в редактор → **Deploy**.
5. Запомните URL воркера, например `https://nutrilens-proxy.<ваш-сабдомен>.workers.dev`.

## Заполнение в приложении (Настройки → «Прокси / обход блокировок»)

| Поле в приложении        | Значение                                              |
|--------------------------|-------------------------------------------------------|
| URL прокси для Gemini    | `https://<worker>.workers.dev/gemini`                 |
| URL прокси для NanoGPT   | `https://<worker>.workers.dev/nano`                   |

Оставьте поле пустым, чтобы приложение подключалось напрямую.

## (Опционально) Защита общим токеном

Чтобы чужие пользователи не расходовали ваш воркер, задайте переменную окружения:

- Вкладка **Settings** → **Variables and Secrets** → **Add**:
  - Name: `PROXY_TOKEN`
  - Value: любой длинный случайный секрет
  - Type: **Secret**

После этого все запросы должны дополнительно содержать заголовок `X-Proxy-Token: <значение>`. В самом приложении этот заголовок пока не отправляется — токен-защита имеет смысл, только если вы доработаете клиент (например, захардкодите его или вынесете в Настройки). Без переменной `PROXY_TOKEN` воркер публичный.

## Проверка

```bash
# Gemini: должен вернуть JSON с ошибкой аутентификации (значит, проксируется)
curl https://<worker>.workers.dev/gemini/v1beta/models

# NanoGPT: должен вернуть JSON с ошибкой аутентификации
curl https://<worker>.workers.dev/nano/api/v1/chat/completions
```

Если вернулся JSON, а не сетевая ошибка — прокси работает.

## Заметки

- Воркер **не логирует** ключи и тела запросов.
- CORS разрешён для всех источников (`*`), чтобы браузер мог обращаться напрямую.
- Лимиты Cloudflare Free: 100 000 запросов/день — для личного использования более чем достаточно.
